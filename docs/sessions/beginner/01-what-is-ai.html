What is AI?

The AI Landscape

The terms "AI," "Machine Learning," "Deep Learning," and "Generative AI" get thrown around interchangeably — but they're not the same thing. They form a nested hierarchy, where each inner field is a more specific subset of the outer one.

Artificial Intelligence
Any technique that enables machines to mimic human intelligence
Machine Learning
Systems that learn from data without being explicitly programmed
Deep Learning
Neural networks with many layers learning hierarchical representations
Generative AI
Models that generate new content: text, images, audio, code

Every LLM is Deep Learning. Every Deep Learning model is ML. Every ML system is AI. But not every AI system uses ML.

Breaking Down Each Layer

Artificial Intelligence (1950s→) is the broad field of creating machines that appear intelligent. Early AI was mostly rule-based: "if the user asks about weather, fetch the forecast." These systems are brittle — they can only handle what their rules explicitly cover.

Machine Learning (1980s→) flipped the script. Instead of writing rules, you show the system thousands of examples and let it discover the patterns itself. A spam filter doesn't have explicit rules for every spam message — it learns from millions of labeled emails what spam looks like.

Deep Learning (2010s→) uses artificial neural networks with many layers ("deep" refers to the depth of these networks). Each layer learns increasingly abstract representations. In image recognition: layer 1 detects edges, layer 2 detects shapes, layer 3 detects objects. This breakthrough enabled dramatically better performance on complex tasks like speech recognition and image classification.

Generative AI (2020s→) is the most recent wave. These deep learning models can generate new content — not just classify inputs. GPT-4, Claude, Stable Diffusion, and Midjourney all fall here. They're trained on internet-scale data and have learned to produce coherent text, images, and code.

Key Insight

When someone says "AI" in 2024, they usually mean Generative AI or a specific ML application. When a researcher says "AI," they might mean the broader field. Context matters — the word AI is genuinely ambiguous.

How AI Makes Decisions

AI systems don't think the way humans do. There's no reasoning, no common sense, no understanding. What they do is find and apply statistical patterns — an idea that sounds simple but enables remarkably powerful behavior.

The Core Loop: Training and Inference

Every ML system goes through two phases:

  • Training: The model is shown millions of examples with correct answers. It adjusts its internal parameters to minimize errors. This can take weeks on thousands of GPUs.
  • Inference: The trained model is given new inputs and produces outputs based on the patterns it learned. This happens in real-time when you use ChatGPT or Google Translate.

Three Concrete Examples

Spam Detection

A spam filter is trained on millions of emails labeled "spam" or "not spam." It learns that phrases like "CLICK HERE NOW," excessive caps, and certain sender patterns correlate with spam. On new emails, it computes a probability score and decides which folder to use — without any explicit rules about what spam looks like.

Image Recognition

A model trained on millions of labeled images (cats, dogs, cars...) learns which pixel patterns correspond to which objects. It doesn't know what a cat "is" — it knows that a certain distribution of edge patterns, colors, and shapes correlates with the label "cat" in its training data.

Language Translation

Google Translate was trained on billions of parallel sentence pairs (the same sentence in two languages). It learns the statistical relationships between word patterns across languages. It doesn't understand grammar rules — it has learned which sequences of words in one language tend to correspond to which sequences in another.

Important to Understand

AI doesn't have intuition — it finds patterns in data it was trained on. When you ask an LLM a factual question, it doesn't "look up" the answer. It generates the most statistically likely next tokens based on its training data. This is why AI can be confidently wrong.

Types of AI Systems

Not all AI is created equal. The type of system matters enormously for understanding its capabilities and failure modes. Here are the three main paradigms you'll encounter:

Type How It Works Strengths Weaknesses Example
Rule-Based Explicit if/then logic written by engineers Predictable, auditable, no training data needed Brittle, can't handle edge cases, doesn't improve Chess engine rules, tax calculator, decision trees
Classical ML Learns statistical patterns from labeled examples Generalizes, improves with more data, explainable models available Needs lots of labeled data, feature engineering required Spam filter, credit scoring, recommendation systems
Large Language Models Transformer neural nets trained on internet-scale text to predict next tokens Incredibly versatile, few-shot learning, handles language nuance Hallucinations, opaque, expensive, training data cutoff ChatGPT, Claude, Gemini, GitHub Copilot

When to Use Which

A common mistake is reaching for LLMs when a simpler rule-based or classical ML approach is better. If you need to validate an email address — use a regex (rule-based). If you need to classify sentiment in millions of product reviews — classical ML may be more efficient. If you need to understand nuanced customer feedback and generate personalized responses — LLMs shine.

Pro Tip

The best AI engineers use the simplest tool that gets the job done. LLMs are powerful but slow and expensive. For structured prediction tasks with labeled data, classical ML is often faster, cheaper, and more reliable.

AI vs Human Intelligence

AI is often framed as either "just a tool" or "about to surpass us." The reality is more nuanced. Current AI systems are superhuman in narrow domains and surprisingly brittle in others.

What AI Does Better Than Humans

  • Scale and speed: Process millions of documents, images, or data points in the time a human could read one.
  • Consistency: Performs the same at 3am as at 9am — no fatigue, no cognitive load effects.
  • Pattern recognition at scale: Spots correlations in high-dimensional data that humans would never find manually.
  • Specific narrow tasks: Beating world champions at Go, chess, protein folding prediction, weather forecasting.

What Humans Do Better (For Now)

  • Common sense reasoning: Understanding that a cup can't hold water if it has a hole in the bottom, without needing millions of examples.
  • Causal reasoning: Understanding that removing a cause prevents an effect, not just correlation.
  • Embodied understanding: Knowing what "hot" feels like, why being late is stressful, why humor is funny.
  • Novel problem solving: Inventing entirely new approaches to problems outside the training distribution.
  • Adaptive learning: Humans learn from very few examples; LLMs need vast data to learn new skills.
Hallucinations: The Critical Limitation

AI systems can confidently produce wrong answers — a phenomenon called "hallucination." An LLM doesn't know what it doesn't know. It will generate a plausible-sounding but completely fabricated citation, statistic, or historical event with the same confident tone as a correct answer. Always verify AI outputs on matters of fact.

The Practical Takeaway

Think of current AI not as artificial general intelligence, but as a very powerful pattern completion engine. It's extraordinary at tasks where the answer is "what would a human expert typically say here?" It struggles when the correct answer requires true reasoning, real-world embodied understanding, or anything significantly outside its training distribution.

The Right Mental Model

The most productive way to think about current AI: it's an extremely well-read, very fast intern who has read everything but experienced nothing, and who is confident even when wrong. Your job is to provide the judgment, verification, and domain context they lack.

    What type of AI is Netflix recommendations?
    Classical ML — Netflix uses collaborative filtering and matrix factorization models trained on user watch history, ratings, and behavior. The model predicts which content each user is likely to enjoy based on patterns from similar users. No generative AI involved — it's predicting preferences, not creating content.
    What type of AI is GitHub Copilot?
    Generative AI — Copilot is powered by OpenAI Codex (a descendant of GPT), which is a large language model fine-tuned on billions of lines of public code. It generates new code completions, not just retrieving cached snippets. Training data includes GitHub repos, documentation, and Stack Overflow.
    What type of AI is Stockfish (chess engine)?
    Rule-Based + Search — Traditional chess engines like Stockfish use hand-crafted evaluation functions (piece values, position heuristics) combined with minimax search with alpha-beta pruning. It doesn't learn from data — the rules were written by expert chess programmers. Note: AlphaZero / Leela Chess Zero DO use ML (self-play reinforcement learning), which is different.
    What type of AI is Apple Face ID?
    Classical ML / Deep Learning — Face ID uses a neural network trained to recognize your face from an infrared dot-grid scan. The model learned to distinguish your unique facial geometry during setup. It's deep learning (a convolutional neural network) but not generative AI — it's classifying "is this the authorized user?" not generating anything.
  4. Reflect on patterns
    After classifying all 20, you should notice a pattern: most recommendation systems are Classical ML; most chatbots and code tools are Generative AI; most rule-based systems are either legacy tech or highly constrained domains (medical devices, avionics). The lines are blurring — many modern systems combine all three approaches.

Why Python for AI?

Python is the undisputed language of AI and machine learning. Over 85% of ML papers, tutorials, and production systems use Python. If you want to work with AI, you need Python — but the good news is you only need a small, practical subset of it to get started.

Why Python Won the AI Wars

  • Readable syntax: Python reads almost like English pseudocode. Lower cognitive overhead means you can focus on AI concepts instead of syntax battles.
  • The ecosystem: Every major AI framework ships Python first — PyTorch, TensorFlow, HuggingFace, LangChain, scikit-learn. Nothing else comes close.
  • Interactive development: Jupyter/Colab notebooks let you run code cell-by-cell, see results instantly, and iterate rapidly — perfect for experimentation.
  • Community: Virtually every AI question has already been answered on Stack Overflow. Virtually every library has tutorials, examples, and active maintainers.

The Python AI Ecosystem

Data
NumPy & Pandas

Arrays, DataFrames, data manipulation. The foundation everything else builds on.

Deep Learning
PyTorch & TensorFlow

Build and train neural networks. PyTorch is the research standard; TF still widely used in production.

Models

50,000+ pre-trained models. Load BERT, Llama, Whisper, CLIP with 3 lines of code.

LLM Frameworks
LangChain & LlamaIndex

Build RAG pipelines, agents, and LLM-powered apps with standardized abstractions.

APIs
openai, anthropic, google-generativeai

Official SDK clients for calling GPT-4, Claude, Gemini from your Python code.

Classic ML
scikit-learn

Linear regression to random forests. The gold standard for non-deep-learning ML.

You Don't Need to Learn All of Python

To build AI applications, you need maybe 20% of Python. Focus on: variables, lists, dicts, functions, loops, and file I/O. The AI frameworks handle the complex stuff. This session covers exactly what you need.

Variables and Types

Python uses dynamic typing — you don't declare types, Python infers them. The types you'll use most in AI work:

basics.py
# === Variables and Types ===

# Strings - the most common type in LLM work
prompt = "Explain machine learning in simple terms"
model_name = "gpt-4o"
api_key = "sk-..."  # Never hardcode real keys!

# Numbers
temperature = 0.7        # float: controls LLM randomness (0.0-2.0)
max_tokens = 1024        # int: limits response length
session_id = 42

# Booleans
is_streaming = True
use_cache = False

# None - represents "no value", like null in other languages
response = None  # Will be set after API call

# Type checking (rarely needed but useful to know)
print(type(prompt))      # 
print(type(temperature)) # 
print(type(max_tokens))  # 

# String formatting - f-strings are your best friend
user_name = "Alice"
session_count = 3
greeting = f"Welcome back, {user_name}! You've completed {session_count} sessions."
print(greeting)
# Output: Welcome back, Alice! You've completed 3 sessions.

# Multi-line strings for long prompts
system_prompt = """
You are a helpful AI assistant.
You respond concisely and accurately.
You never make up facts.
"""
print(system_prompt.strip())  # .strip() removes leading/trailing whitespace

Lists — Sequences of Things

Lists are ordered, mutable collections. In AI work, you'll use lists constantly: message histories, token sequences, batch inputs, search results.

basics.py
# === Lists ===

# A list of model names to try
models = ["gpt-4o", "gpt-4o-mini", "claude-3-5-sonnet", "gemini-1.5-pro"]

# Accessing elements (0-indexed)
print(models[0])   # "gpt-4o"  (first element)
print(models[-1])  # "gemini-1.5-pro"  (last element)

# Adding and removing
models.append("llama-3.1-70b")   # add to end
models.insert(0, "o1-preview")   # insert at position 0
models.remove("gpt-4o-mini")     # remove by value

# Slicing: get a subset [start:stop] (stop is exclusive)
first_two = models[:2]    # ["o1-preview", "gpt-4o"]
last_two = models[-2:]    # last 2 elements

# Iterating - how you'll process AI outputs
for model in models:
    print(f"Testing model: {model}")

# Length
print(f"Testing {len(models)} models")

# The messages array: THE most important list in LLM work
messages = [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user",   "content": "What is machine learning?"},
]
# Add the assistant's response after calling the API
messages.append({
    "role": "assistant",
    "content": "Machine learning is a subset of AI..."
})
# Then add the next user message to continue the conversation
messages.append({
    "role": "user",
    "content": "Can you give me an example?"
})
print(f"Conversation has {len(messages)} messages")

Dictionaries — Structured Data

Dictionaries store key-value pairs. API responses, model configs, and structured outputs are almost always dicts.

basics.py
# === Dictionaries ===

# Model configuration as a dict
config = {
    "model": "gpt-4o",
    "temperature": 0.7,
    "max_tokens": 1024,
    "stream": False
}

# Accessing values
print(config["model"])           # "gpt-4o"
print(config.get("timeout", 30)) # .get() with default value if key missing

# Modifying
config["temperature"] = 0.3     # update
config["top_p"] = 0.9            # add new key
del config["stream"]             # delete

# Nested dicts - very common in API responses
api_response = {
    "id": "chatcmpl-abc123",
    "model": "gpt-4o",
    "usage": {
        "prompt_tokens": 156,
        "completion_tokens": 84,
        "total_tokens": 240
    },
    "choices": [
        {
            "message": {
                "role": "assistant",
                "content": "Machine learning is..."
            },
            "finish_reason": "stop"
        }
    ]
}

# Navigating nested structure
response_text = api_response["choices"][0]["message"]["content"]
total_tokens = api_response["usage"]["total_tokens"]
print(f"Response: {response_text[:50]}...")
print(f"Used {total_tokens} tokens")

# Iterating over a dict
for key, value in config.items():
    print(f"{key}: {value}")

Working with Functions

Functions let you package reusable logic. In AI work, you'll write functions to call APIs, process responses, format prompts, and handle errors. Modern Python uses type hints to document what types a function expects and returns.

functions.py
from typing import Optional

# Basic function with type hints
# Parameters: name and type -> return type after the arrow
def build_prompt(
    task: str,
    context: str,
    output_format: str = "bullet points",  # default value
    max_words: Optional[int] = None         # Optional means it can be None
) -> str:
    """
    Build a structured prompt from components.

    Args:
        task: What you want the LLM to do
        context: Background information
        output_format: How to structure the response
        max_words: Optional word limit for response

    Returns:
        A formatted prompt string
    """
    prompt_parts = [
        f"Task: {task}",
        f"Context: {context}",
        f"Output format: {output_format}",
    ]

    if max_words is not None:
        prompt_parts.append(f"Constraint: Keep response under {max_words} words.")

    return "\n".join(prompt_parts)


# Function to safely extract text from an OpenAI-style response
def extract_response_text(api_response: dict) -> str:
    """Extract the assistant's message text from an API response dict."""
    try:
        return api_response["choices"][0]["message"]["content"]
    except (KeyError, IndexError) as e:
        print(f"Warning: Could not extract response text: {e}")
        return ""


# Function with a list return type
def count_tokens_estimate(text: str) -> int:
    """
    Rough token count estimate (actual tokenization varies by model).
    Rule of thumb: ~4 characters per token for English text.
    """
    return len(text) // 4


# Using the functions
prompt = build_prompt(
    task="Explain the difference between supervised and unsupervised learning",
    context="The reader is a software developer with no ML background",
    output_format="Two paragraphs, one for each type",
    max_words=150
)
print(prompt)
print(f"\nEstimated tokens: {count_tokens_estimate(prompt)}")
Type Hints Are Documentation, Not Enforcement

Python doesn't enforce type hints at runtime by default — they're hints for developers and tools like mypy or your IDE. But using them consistently makes your AI code dramatically more readable and maintainable, especially when you're building prompt pipelines with many interconnected functions.

File I/O and APIs

AI applications need to read data from files and communicate with external APIs. Here's the essential toolkit.

Reading JSON Files

JSON (JavaScript Object Notation) is the universal data format for APIs and configuration. Python's built-in json module handles it natively.

file_io.py
import json

# Reading JSON from a file
with open("sessions.json", "r") as f:
    sessions = json.load(f)  # parse JSON → Python list/dict

print(f"Loaded {len(sessions)} sessions")
print(sessions[0])  # {'id': 1, 'title': 'What is AI?', ...}

# Writing JSON to a file
results = {
    "model": "gpt-4o",
    "responses": [
        {"question": "What is AI?", "answer": "AI is...", "tokens": 42}
    ]
}
with open("results.json", "w") as f:
    json.dump(results, f, indent=2)  # indent=2 for pretty printing

# Converting between JSON string and Python dict
json_string = '{"role": "user", "content": "Hello!"}'
message_dict = json.loads(json_string)   # string → dict
back_to_string = json.dumps(message_dict, indent=2)  # dict → string

Making API Requests

The requests library is the standard for HTTP calls in Python. Install with: pip install requests

api_call.py
import requests
import os
import json

# Load API key from environment variable (NEVER hardcode it)
api_key = os.environ.get("OPENAI_API_KEY")
if not api_key:
    raise ValueError("OPENAI_API_KEY environment variable not set")

# The OpenAI Chat Completions API endpoint
url = "https://api.openai.com/v1/chat/completions"

# Request headers (authentication)
headers = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json"
}

# Request body (what we're asking for)
body = {
    "model": "gpt-4o-mini",           # cheaper model for testing
    "temperature": 0.7,
    "max_tokens": 500,
    "messages": [
        {
            "role": "system",
            "content": "You are a concise AI tutor."
        },
        {
            "role": "user",
            "content": "What is the difference between AI and ML in 2 sentences?"
        }
    ]
}

# Make the POST request
response = requests.post(url, headers=headers, json=body)

# Check for errors
if response.status_code != 200:
    print(f"Error {response.status_code}: {response.text}")
else:
    # Parse the JSON response
    data = response.json()

    # Extract the text response
    answer = data["choices"][0]["message"]["content"]
    tokens_used = data["usage"]["total_tokens"]

    print(f"Answer: {answer}")
    print(f"Tokens used: {tokens_used}")

    # Tip: Use the official openai Python package in practice
    # pip install openai
    # It handles auth, retries, and parsing automatically
Never Hardcode API Keys

API keys are like passwords — never write them directly in your code. Always use environment variables (os.environ.get("KEY_NAME")) or a .env file with the python-dotenv package. If you accidentally commit a key to GitHub, rotate it immediately — bots scrape GitHub for keys in seconds.

Python for Data: Quick Intro

Before you can train or evaluate AI models, you need to load and understand your data. pandas is the go-to library for tabular data in Python — think Excel but programmable and scalable to millions of rows.

Install with: pip install pandas

pandas_intro.py
import pandas as pd

# === Loading a CSV file ===
# Let's say we have a dataset of customer support tickets
df = pd.read_csv("support_tickets.csv")

# Always start by inspecting what you loaded
print("Shape (rows, columns):", df.shape)
# Output: Shape (rows, columns): (5000, 8)

# View the first 5 rows
print(df.head())
#    ticket_id  category  priority  status    response_time_mins  ...
# 0  T001       billing   high      resolved  12.5
# 1  T002       tech      medium    open      NaN
# ...

# View last 5 rows
print(df.tail())

# Column names and data types
print(df.dtypes)
# ticket_id           object
# category            object
# priority            object
# response_time_mins  float64
# ...

# === Summary Statistics ===
# For numeric columns: count, mean, std, min, max, quartiles
print(df.describe())
#        response_time_mins
# count  4823.000000         (note: 5000 - 177 NaN values)
# mean   24.3
# std    18.7
# min    0.5
# 25%    9.0
# 50%    18.0   (median)
# 75%    35.0

high_priority = df[df["priority"] == "high"]

print(avg_by_category)

# tech       31.7

# Value counts: how many of each category?

# === Missing values ===

# === Saving results ===
high_priority.to_csv("high_priority_tickets.csv", index=False)

Connecting Data to AI

In practice, you'll load a dataset with pandas, then feed it to an LLM or ML model. A common pattern is to summarize statistics, format them as text, and include them in a prompt:

data_to_llm.py
import pandas as pd

# Load data
df = pd.read_csv("support_tickets.csv")

# Create a text summary of key statistics
summary = f"""
Dataset: Customer Support Tickets
Total tickets: {len(df):,}
Date range: {df['date'].min()} to {df['date'].max()}

By status:
{df['status'].value_counts().to_string()}

Average response time: {df['response_time_mins'].mean():.1f} minutes
Tickets over 1 hour: {(df['response_time_mins'] > 60).sum()} ({(df['response_time_mins'] > 60).mean():.1%})

Top categories:
{df['category'].value_counts().head(5).to_string()}
"""

# Now build a prompt with this data summary
prompt = f"""
You are a customer experience analyst.

Analyze this support ticket dataset and identify:
1. The 3 most pressing issues
2. Which categories need more resources
3. One actionable recommendation

Dataset Summary:
{summary}

Keep your analysis under 200 words. Be specific with numbers.
"""

print(prompt)
# Ready to send this to any LLM API!
The Pattern You'll Use Constantly

Load data → compute statistics → format as text → include in prompt → call LLM API → parse response → act on results. This loop powers most practical AI applications. You've now learned all the Python pieces you need to execute it.

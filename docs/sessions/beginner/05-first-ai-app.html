The AI App Architecture

The architecture of a modern AI app is simpler than most people expect. You don't need to train a model, provision GPUs, or understand the math. The core loop is just four steps:

User Input
→
Prompt Template
→
LLM API
→
Response

The prompt template is where your application logic lives. It combines a system prompt (the AI's instructions) with the user's message and any relevant context. The API handles the hard work — you just send text and receive text back.

Key Insight: You don't need to train a model. You can call existing, state-of-the-art models like GPT-4o or Claude via API. This is how 95% of production AI apps are built today.

The OpenAI API

The OpenAI Chat Completions API takes a list of messages and returns the model's next message. Each message has a role and content. There are three roles:

  • system — Sets the AI's persona, tone, and behavioral rules. Read once at the start. This is your main control lever.
  • user — What the human says or asks.
  • assistant — What the AI has said in previous turns (for multi-turn conversations).

Two other parameters you'll always configure:

  • temperature (0–2): Controls randomness. 0 = deterministic and focused. 1 = balanced. 2 = highly creative and unpredictable. For factual tasks, use 0–0.3. For creative tasks, use 0.7–1.2.
  • max_tokens: Hard cap on response length. 1 token ≈ 4 characters. A typical paragraph is ~100–150 tokens.
api_chat.py 
import os
from openai import OpenAI

# The client reads OPENAI_API_KEY from environment automatically
client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o-mini",          # cheaper model, great for learning
    temperature=0.7,               # balanced creativity
    max_tokens=500,                # limit response length
    messages=[
        {
            "role": "system",
            "content": "You are a helpful AI tutor specializing in explaining "
                       "complex topics in simple, clear language. Always use "
                       "concrete examples and analogies."
        },
        {
            "role": "user",
            "content": "What is gradient descent? Explain it like I'm 15."
        }
    ]
)

# Extract the text from the response
answer = response.choices[0].message.content
print(answer)

# You can also inspect token usage (for cost tracking)
print(f"\nTokens used: {response.usage.total_tokens}")

Building a Study Assistant

A study assistant chatbot has one job: help the user understand topics on demand. The system prompt is where we define this behavior, set the tone, and add any constraints. The application loop reads user input, builds the message list, calls the API, and prints the reply.

Notice how we maintain a conversation_history list. This is how you give the model "memory" — you include prior messages in every API call. The model is stateless; it only knows what you send it.

study_assistant.py 
import os
from openai import OpenAI

client = OpenAI()   # reads OPENAI_API_KEY from environment

SYSTEM_PROMPT = """You are an expert AI study assistant helping a student
learn machine learning and AI concepts from scratch.

Your rules:
- Explain concepts clearly and concisely using everyday analogies
- When asked for examples, give 2-3 concrete ones
- If the student seems confused, try a different explanation angle
- End each response with one follow-up question to check understanding
- Keep responses under 200 words unless asked for more detail
"""

def chat(user_message: str, history: list) -> str:
    """Send a message and get a reply, maintaining conversation history."""
    history.append({"role": "user", "content": user_message})

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        temperature=0.6,
        max_tokens=600,
        messages=[{"role": "system", "content": SYSTEM_PROMPT}] + history
    )

    assistant_reply = response.choices[0].message.content
    history.append({"role": "assistant", "content": assistant_reply})
    return assistant_reply


def main():
    print("Study Assistant ready. Type 'quit' to exit.\n")
    history = []

    while True:
        user_input = input("You: ").strip()
        if not user_input or user_input.lower() in ("quit", "exit"):
            print("Goodbye!")
            break

        reply = chat(user_input, history)
        print(f"\nAssistant: {reply}\n")


if __name__ == "__main__":
    main()

Error Handling & Best Practices

Production AI apps need to handle failures gracefully. The most common issues are rate limits (too many requests per minute), network timeouts, and authentication errors. Wrap your API calls in try/except blocks and implement exponential backoff for rate limit errors.

robust_api_call.py 
import os
import time
from openai import OpenAI, RateLimitError, APITimeoutError, APIConnectionError

client = OpenAI()

def call_with_retry(messages: list, max_retries: int = 3) -> str:
    """Call the API with exponential backoff on rate limit errors."""
    for attempt in range(max_retries):
        try:
            response = client.chat.completions.create(
                model="gpt-4o-mini",
                messages=messages,
                timeout=30.0       # don't wait more than 30 seconds
            )
            return response.choices[0].message.content

        except RateLimitError:
            wait = 2 ** attempt    # 1s, 2s, 4s
            print(f"Rate limited. Waiting {wait}s before retry {attempt + 1}/{max_retries}")
            time.sleep(wait)

        except APITimeoutError:
            print(f"Request timed out (attempt {attempt + 1})")

        except APIConnectionError as e:
            print(f"Connection error: {e}")
            break

    return "Sorry, the AI service is temporarily unavailable. Please try again."

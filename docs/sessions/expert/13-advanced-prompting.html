Prompt Chains

Instead of one giant prompt, break complex tasks into a sequence where the output of each step is the input to the next. Chains are more reliable, debuggable, and composable.

import anthropic

client = anthropic.Anthropic()

def call(prompt: str, system: str = "") -> str:
    msg = client.messages.create(
        model="claude-opus-4-7",
        max_tokens=1024,
        system=system,
        messages=[{"role": "user", "content": prompt}]
    )
    return msg.content[0].text

# Chain: raw text → key facts → structured summary → action items
def research_chain(raw_text: str) -> dict:
    # Step 1: Extract key facts
    facts = call(
        f"Extract the 5 most important factual claims from this text. Return as a numbered list.\n\n{raw_text}"
    )

    # Step 2: Synthesize into structured summary
    summary = call(
        f"Given these facts:\n{facts}\n\nWrite a 3-sentence executive summary.",
        system="You are a business analyst. Be concise and actionable."
    )

    # Step 3: Generate action items from the summary
    actions = call(
        f"Based on this summary:\n{summary}\n\nList 3 concrete next steps a manager should take.",
        system="You produce short, specific action items. No fluff."
    )

    return {"facts": facts, "summary": summary, "actions": actions}

result = research_chain("Q3 revenue grew 23% to $4.2M. Customer churn increased to 8%. Three key deals slipped into Q4. The new product line launched in September contributed $800K.")
print(result["actions"])

System Prompt Engineering

The system prompt is your single most powerful lever for controlling model behavior. It sets persona, constraints, output format, and guardrails that apply to every turn.

✨

System Prompt Anatomy

A complete system prompt has: (1) Persona — who the model is. (2) Task — what it does. (3) Constraints — what it must never do. (4) Format — how it structures output. (5) Examples — golden in/out pairs for tricky cases.

SYSTEM_PROMPT = """You are a customer support agent for Acme SaaS, a project management platform.

## Your role
- Answer questions about Acme SaaS features, pricing, and troubleshooting
- Escalate billing disputes and security issues to human agents
- Stay friendly, concise, and factual

## Constraints
- NEVER discuss competitor products by name
- NEVER make promises about future features ("coming soon" is okay)
- NEVER access or reference user account data — you don't have it
- If asked something outside your scope, say: "That's outside my area — let me connect you with the right team."

## Output format
- Use bullet points for step-by-step instructions
- Keep responses under 200 words
- End troubleshooting responses with: "Did that resolve your issue? [Yes] [No]"

## Example
User: "How do I export my project?"
You: "To export your project:\n• Go to Settings → Export\n• Choose CSV or JSON\n• Click Download\n\nDid that resolve your issue? [Yes] [No]"
"""

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=300,
    system=SYSTEM_PROMPT,
    messages=[{"role": "user", "content": "How do I add a new team member?"}]
)
print(response.content[0].text)

Guardrails and Output Validation

Production AI needs guardrails — post-processing checks that catch bad outputs before they reach users. Never trust raw LLM output in production.

import json
import re

def extract_json(text: str) -> dict | None:
    """Robustly extract JSON from LLM output even if wrapped in markdown."""
    # Try parsing directly
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        pass
    # Try extracting from code block
    match = re.search(r'```(?:json)?\s*(\{.*?\})\s*```', text, re.DOTALL)
    if match:
        try:
            return json.loads(match.group(1))
        except json.JSONDecodeError:
            pass
    return None

def safe_classify(text: str) -> dict:
    prompt = f"""Classify this customer message. Return ONLY valid JSON with keys:
"intent" (one of: billing, technical, feature_request, other),
"urgency" (1-5),
"sentiment" (positive/neutral/negative)

Message: {text}"""

    response = call(prompt)
    result = extract_json(response)

    # Validate the output
    if not result:
        return {"intent": "other", "urgency": 1, "sentiment": "neutral", "error": "parse_failed"}

    valid_intents = {"billing", "technical", "feature_request", "other"}
    if result.get("intent") not in valid_intents:
        result["intent"] = "other"

    urgency = result.get("urgency", 1)
    if not isinstance(urgency, int) or not 1 <= urgency <= 5:
        result["urgency"] = 3

    return result

result = safe_classify("My card was charged twice this month and I'm furious!")
print(result)  # {"intent": "billing", "urgency": 5, "sentiment": "negative"}

Advanced Few-Shot Strategies

Beyond simple examples: select examples dynamically, use negative examples, and structure examples for maximum signal.

def build_dynamic_few_shot(query: str, example_bank: list[dict], top_k: int = 3) -> str:
    """Select most-relevant examples for the current query using keyword overlap."""
    query_words = set(query.lower().split())

    scored = []
    for ex in example_bank:
        ex_words = set(ex["input"].lower().split())
        overlap = len(query_words & ex_words)
        scored.append((overlap, ex))

    top_examples = [ex for _, ex in sorted(scored, reverse=True)[:top_k]]

    examples_text = "\n\n".join([
        f"Input: {e['input']}\nOutput: {e['output']}"
        for e in top_examples
    ])

    return f"""Here are examples of how to handle similar requests:

{examples_text}

Now handle this:
Input: {query}
Output:"""

example_bank = [
    {"input": "Cancel my subscription", "output": "INTENT: cancellation | URGENCY: high | ROUTE: retention_team"},
    {"input": "Invoice PDF download", "output": "INTENT: billing | URGENCY: low | ROUTE: self_serve"},
    {"input": "App crashes on export", "output": "INTENT: bug_report | URGENCY: high | ROUTE: engineering"},
    {"input": "How to add users", "output": "INTENT: how_to | URGENCY: low | ROUTE: docs_bot"},
]

prompt = build_dynamic_few_shot("I can't download my invoice", example_bank)
print(call(prompt))

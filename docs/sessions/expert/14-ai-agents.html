What Makes an Agent?

An agent is an LLM with tools, a goal, and a loop. Unlike a one-shot prompt, an agent can take actions, observe results, and adapt its plan until the task is done.

💬

Chatbot (not an agent)

Responds to one message at a time. No tools. No persistent state. Can't take actions in the real world. Task ends when it outputs text.

🤖

Agent

Has tools (search, code execution, API calls). Runs in a loop until goal is achieved. Can plan sub-tasks, use memory, and coordinate with other agents.

Tool Calling

The Anthropic API supports native tool calling: you define tools with JSON schemas, the model decides when to call them, you execute the call and return results, the model continues.

import anthropic
import json

client = anthropic.Anthropic()

# Define tools the agent can use
tools = [
    {
        "name": "search_web",
        "description": "Search the web for current information on a topic.",
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {"type": "string", "description": "The search query"}
            },
            "required": ["query"]
        }
    },
    {
        "name": "calculate",
        "description": "Perform a mathematical calculation.",
        "input_schema": {
            "type": "object",
            "properties": {
                "expression": {"type": "string", "description": "Math expression to evaluate, e.g. '2 * (3 + 4)'"}
            },
            "required": ["expression"]
        }
    }
]

# Mock tool execution (replace with real implementations)
def execute_tool(name: str, inputs: dict) -> str:
    if name == "search_web":
        return f"[Search results for '{inputs['query']}']: Python 3.12 released October 2023, added type aliases, improved error messages..."
    if name == "calculate":
        try:
            return str(eval(inputs["expression"], {"__builtins__": {}}))
        except Exception as e:
            return f"Error: {e}"
    return "Tool not found"

def run_agent(user_message: str) -> str:
    messages = [{"role": "user", "content": user_message}]

    while True:
        response = client.messages.create(
            model="claude-opus-4-7",
            max_tokens=1024,
            tools=tools,
            messages=messages
        )

        # Agent is done — return the final text response
        if response.stop_reason == "end_turn":
            return response.content[0].text

        # Agent wants to call a tool
        if response.stop_reason == "tool_use":
            # Add the assistant's response (including tool_use blocks) to messages
            messages.append({"role": "assistant", "content": response.content})

            # Execute all tool calls and collect results
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    result = execute_tool(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result
                    })

            # Add tool results as a user message
            messages.append({"role": "user", "content": tool_results})
        else:
            break  # Unexpected stop reason

    return "Agent loop ended unexpectedly"

answer = run_agent("What's 15% of the GDP of Germany? Search for Germany's GDP first.")
print(answer)

Agent Memory

Agents need memory across turns. Four types, each with different trade-offs:

Memory TypeStorageScopeUse Case
In-contextMessages arrayCurrent sessionConversation history, recent tool results
External (vector)Vector DBPersistentLong-term knowledge, user preferences
External (key-value)Redis / DBPersistentFacts, user profile, session state
Cached summaryCompressed contextSessionLong conversations — summarize old turns
 
# Simple key-value memory for an agent
class AgentMemory:
    def __init__(self):
        self._store = {}

    def remember(self, key: str, value: str):
        self._store[key] = value

    def recall(self, key: str) -> str | None:
        return self._store.get(key)

    def as_context(self) -> str:
        if not self._store:
            return ""
        items = "\n".join(f"- {k}: {v}" for k, v in self._store.items())
        return f"Known facts:\n{items}\n\n"

memory = AgentMemory()
memory.remember("user_name", "Alex")
memory.remember("preferred_language", "Python")
memory.remember("last_project", "sentiment analysis pipeline")

system = memory.as_context() + "You are a helpful coding assistant."
print(system)

Multi-Agent Systems

Complex tasks benefit from specialist agents orchestrated by a coordinator. Each agent has a focused role, its own tools, and its own system prompt.

⚠️

When to Use Multi-Agent

Multi-agent adds complexity — only use it when tasks are genuinely parallelizable OR require different skill sets (e.g., a research agent + a writing agent + a fact-checker agent). For simple sequential tasks, a single agent with a chain is simpler and cheaper.

def research_agent(topic: str) -> str:
    """Specialist: finds facts about a topic."""
    response = client.messages.create(
        model="claude-opus-4-7",
        max_tokens=512,
        system="You are a research specialist. Find key facts, statistics, and examples. Be factual, no fluff.",
        messages=[{"role": "user", "content": f"Research: {topic}"}]
    )
    return response.content[0].text

def writer_agent(research: str, format: str) -> str:
    """Specialist: turns research into polished prose."""
    response = client.messages.create(
        model="claude-opus-4-7",
        max_tokens=512,
        system=f"You are an expert technical writer. Transform research into {format}. Engaging but accurate.",
        messages=[{"role": "user", "content": f"Research:\n{research}\n\nWrite a {format}."}]
    )
    return response.content[0].text

def orchestrator(topic: str, output_format: str) -> str:
    """Coordinator: runs the multi-agent pipeline."""
    print(f"[Orchestrator] Researching: {topic}")
    facts = research_agent(topic)

    print(f"[Orchestrator] Writing {output_format}")
    article = writer_agent(facts, output_format)

    return article

result = orchestrator("transformer attention mechanisms", "blog post introduction paragraph")
print(result)

What is Multimodal AI?

Early AI models were unimodal — a text model processed text, an image classifier processed images, and never the twain shall meet. Multimodal AI breaks that boundary, allowing a single model to understand and reason across multiple types of data: text, images, audio, video, and structured data. This is much closer to how humans naturally perceive the world — we read a caption while looking at a photo, we hear someone speak while watching their expressions, we interpret a chart alongside its written description.

The multimodal revolution accelerated dramatically with models like GPT-4V (vision), Claude 3 (image understanding), and Gemini 1.5 Pro (text, image, audio, video in one model). These are no longer research curiosities — they're production-grade APIs you can call right now.

Vision Language Models

Vision Language Models (VLMs) are the workhorse of image + text AI. They combine two components: an image encoder that converts pixels into a dense embedding representation, and a language model that reasons over those embeddings alongside text tokens.

Architecture: How VLMs Work

The image encoder (typically a Vision Transformer, ViT) splits the image into patches (e.g., 16×16 pixel tiles), encodes each patch as a vector, and produces a sequence of visual embeddings. These are projected into the same dimensional space as text token embeddings via a trainable linear projection or cross-attention layer, then concatenated with the text token sequence. The LLM then performs self-attention over the combined visual + text token sequence.

CLIP (Contrastive Language-Image Pretraining) took this further by training the image encoder jointly with a text encoder using contrastive learning on 400M image-caption pairs, learning a shared embedding space where "a photo of a dog" and an image of a dog are near each other.

Use Cases

  • Document understanding: Extract data from scanned invoices, forms, receipts — even handwritten ones.
  • Medical imaging: Describe X-rays, pathology slides, or retinal scans in natural language (always with physician oversight).
  • Accessibility: Generate detailed alt-text for images for screen reader users.
  • Quality control: Inspect product photos for defects on manufacturing lines.
  • Retail: Visual search ("find me something that looks like this") and product attribute extraction.
  • Education: Explain diagrams, charts, and scientific figures in textbooks.

Calling a Vision API

vision_api.py
import base64
import httpx
from openai import OpenAI

client = OpenAI()

def analyze_image(image_path: str, question: str) -> str:
    """
    Send a local image to GPT-4o and ask a question about it.
    Images must be base64-encoded for the API.
    """
    # Read and encode the image
    with open(image_path, "rb") as f:
        image_data = base64.standard_b64encode(f.read()).decode("utf-8")

    # Determine MIME type from extension
    ext = image_path.rsplit(".", 1)[-1].lower()
    mime_map = {"jpg": "image/jpeg", "jpeg": "image/jpeg",
                "png": "image/png", "gif": "image/gif", "webp": "image/webp"}
    mime_type = mime_map.get(ext, "image/jpeg")

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "image_url",
                        "image_url": {
                            "url": f"data:{mime_type};base64,{image_data}",
                            "detail": "high"  # "low" for faster/cheaper, "high" for detailed analysis
                        }
                    },
                    {
                        "type": "text",
                        "text": question
                    }
                ]
            }
        ],
        max_tokens=1024
    )
    return response.choices[0].message.content

# Example: analyze a receipt
result = analyze_image(
    "receipt.jpg",
    "Extract all line items with quantities and prices as JSON."
)
print(result)

# You can also pass a public URL directly (no base64 needed):
def analyze_url_image(url: str, question: str) -> str:
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": url}},
                {"type": "text", "text": question}
            ]
        }]
    )
    return response.choices[0].message.content

Audio AI

Audio modality covers a wide range of AI tasks: converting speech to text (ASR), converting text to speech (TTS), identifying who is speaking (speaker diarization), detecting audio events, and generating music. For most practical AI engineering tasks, the most important audio models are Whisper (speech-to-text) and TTS systems.

Whisper: Speech-to-Text

OpenAI's Whisper was trained on 680,000 hours of multilingual audio from the web, making it one of the most robust speech recognition models available. Key characteristics:

  • Supports 99 languages with automatic language detection.
  • Handles accents, noise, technical jargon, and code-switching well.
  • Available as an API endpoint or as an open-source model you can self-host.
  • Multiple model sizes: tiny (39M params, fast), base, small, medium, large-v3 (1.5B params, most accurate).
whisper_transcribe.py
from openai import OpenAI
from pathlib import Path

client = OpenAI()

def transcribe_audio(
    audio_path: str,
    language: str = None,
    response_format: str = "verbose_json"
) -> dict:
    """
    Transcribe audio using Whisper API.
    verbose_json includes timestamps, which are useful for chunking long audio.
    """
    with open(audio_path, "rb") as audio_file:
        kwargs = {
            "model": "whisper-1",
            "file": audio_file,
            "response_format": response_format,
        }
        if language:
            kwargs["language"] = language  # e.g., "en", "es", "fr"

        transcript = client.audio.transcriptions.create(**kwargs)

    return transcript

# Basic transcription
result = transcribe_audio("meeting_recording.mp3")
print(result.text)  # Full transcript

# With timestamps (useful for long recordings)
result = transcribe_audio("meeting_recording.mp3", response_format="verbose_json")
for segment in result.segments:
    start = segment["start"]
    end = segment["end"]
    text = segment["text"]
    print(f"[{start:.1f}s – {end:.1f}s] {text}")

# Self-hosted Whisper (faster for large volumes, avoids API costs)
# pip install openai-whisper
import whisper

model = whisper.load_model("large-v3")  # Downloads on first use
result = model.transcribe("meeting_recording.mp3", language="en")
print(result["text"])

Text-to-Speech

tts_example.py
from openai import OpenAI
from pathlib import Path

client = OpenAI()

def text_to_speech(
    text: str,
    output_path: str = "output.mp3",
    voice: str = "alloy",       # alloy, echo, fable, onyx, nova, shimmer
    model: str = "tts-1",       # tts-1 (fast) or tts-1-hd (higher quality)
    speed: float = 1.0          # 0.25 to 4.0
) -> str:
    """Generate speech from text using OpenAI TTS."""
    response = client.audio.speech.create(
        model=model,
        voice=voice,
        input=text,
        speed=speed,
        response_format="mp3"
    )
    response.stream_to_file(output_path)
    return output_path

# Generate audio
text_to_speech(
    "Welcome to the AI course. Today we're covering multimodal AI systems.",
    voice="nova",
    model="tts-1-hd"
)

Speaker Diarization

Speaker diarization answers the question "who spoke when?" — essential for meeting transcription with multiple participants. The open-source pyannote.audio library is the standard tool, and it can be combined with Whisper to produce a fully attributed transcript:

  • Step 1 — Whisper produces a timestamped transcript.
  • Step 2 — Pyannote identifies speaker segments (SPEAKER_00, SPEAKER_01, etc.) with timestamps.
  • Step 3 — Align the two using timestamp overlap to assign speaker labels to each transcript segment.

Use Cases for Audio AI

  • Meeting transcription and summarization: Record → transcribe → summarize → extract action items.
  • Podcast search: Transcribe episodes, embed chunks, enable semantic search across thousands of hours.
  • Voicebots and IVR: STT → LLM → TTS pipeline for phone-based customer service.
  • Accessibility: Real-time captions for live events, video content.
  • Language learning: Pronunciation feedback, listening comprehension exercises.

Video AI

Video is the most data-rich and computationally demanding modality. A single second of 1080p video at 30fps contains 30 images, each of which might take 500–1000 tokens to represent in a VLM. A 1-hour meeting is 108,000 frames — far beyond any current context window if processed naively.

Frame-Based Analysis

The most practical approach for current LLMs is to treat video as a sampled sequence of still images. You choose a sampling rate (e.g., 1 frame per second, or 1 frame per 10 seconds for slow-moving content), extract keyframes, and send them to a VLM. This works well for:

  • Slide deck recordings (slides change rarely — 1 fps is more than enough).
  • Security camera footage (motion detection can trigger higher-rate sampling).
  • Tutorial videos (describe what's happening step by step).
video_analysis.py
import cv2
import base64
from openai import OpenAI

client = OpenAI()

def extract_frames(video_path: str, fps: float = 1.0) -> list[str]:
    """Extract frames from video at specified rate, return as base64 strings."""
    cap = cv2.VideoCapture(video_path)
    video_fps = cap.get(cv2.CAP_PROP_FPS)
    frame_interval = int(video_fps / fps)

    frames = []
    frame_count = 0

    while cap.isOpened():
        ret, frame = cap.read()
        if not ret:
            break
        if frame_count % frame_interval == 0:
            # Encode frame as JPEG and convert to base64
            _, buffer = cv2.imencode(".jpg", frame, [cv2.IMWRITE_JPEG_QUALITY, 85])
            b64 = base64.standard_b64encode(buffer).decode("utf-8")
            frames.append(b64)
        frame_count += 1

    cap.release()
    return frames

def analyze_video_segment(video_path: str, question: str, sample_fps: float = 0.5) -> str:
    """Analyze a video by sampling frames and sending them to GPT-4o."""
    frames = extract_frames(video_path, fps=sample_fps)

    # Limit to 20 frames to stay within token budget
    if len(frames) > 20:
        step = len(frames) // 20
        frames = frames[::step][:20]

    content = []
    for i, frame_b64 in enumerate(frames):
        content.append({
            "type": "image_url",
            "image_url": {
                "url": f"data:image/jpeg;base64,{frame_b64}",
                "detail": "low"  # Use "low" for video frames to save tokens
            }
        })

    content.append({"type": "text", "text": question})

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": content}],
        max_tokens=1024
    )
    return response.choices[0].message.content

# Analyze a short video
summary = analyze_video_segment(
    "product_demo.mp4",
    "Describe what happens in this video step by step.",
    sample_fps=0.5
)
print(summary)

Long Video with Gemini 1.5 Pro

Google's Gemini 1.5 Pro has a 1-million-token context window and native video understanding — you can upload a video file directly and ask questions about it without frame extraction. It can process up to 1 hour of video at 1fps natively. The Files API handles the upload asynchronously:

gemini_video.py
import time
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

def analyze_long_video(video_path: str, question: str) -> str:
    """Upload video to Gemini Files API and analyze it."""
    # Upload the video file (handles large files)
    print("Uploading video...")
    video_file = genai.upload_file(path=video_path)

    # Wait for processing to complete
    while video_file.state.name == "PROCESSING":
        print(".", end="", flush=True)
        time.sleep(5)
        video_file = genai.get_file(video_file.name)

    if video_file.state.name == "FAILED":
        raise ValueError(f"Video processing failed: {video_file.state.name}")

    print(f"\nVideo ready: {video_file.uri}")

    model = genai.GenerativeModel("gemini-1.5-pro-latest")
    response = model.generate_content([video_file, question])
    return response.text

result = analyze_long_video(
    "one_hour_meeting.mp4",
    "Summarize the key decisions made and list all action items with owners."
)
Computational Cost

Video AI is computationally expensive — think carefully about sampling rate and clip length. At 1fps, 1 hour of video = 3,600 images. At ~500 tokens per image, that's 1.8M tokens per query. Sample at the lowest rate that still gives you the information you need.

Video AI Use Cases

  • Sports analysis: Detect plays, track player positions, generate highlight reels automatically.
  • Content moderation: Scan user-uploaded video for policy violations at scale.
  • Surveillance: Detect unusual events in security footage without 24/7 human monitoring.
  • Training data labeling: Auto-label video frames for computer vision model training.
  • Video accessibility: Generate audio descriptions for visually impaired viewers.

Building Multimodal Apps

Building apps that combine modalities requires thinking about pipelines — data flows through multiple transformation stages, each involving a different modality. The key is to be explicit about what data lives where and what format conversions happen at each stage.

Common Pipeline Patterns

  • Image → Text → Action: User uploads photo → VLM extracts structured data → app takes action (e.g., receipt photo → expense entry).
  • Audio → Text → Summary: Recording → Whisper transcript → LLM summary + action items.
  • Document → OCR + VLM → Structured Data: Scanned form → Vision API extracts fields → JSON output for downstream systems.
  • Text → Search → Image → Caption: User query → retrieve relevant images → VLM captions them for context.

Complete Image Analysis Pipeline

document_pipeline.py
import base64
import json
from pathlib import Path
from openai import OpenAI
from pydantic import BaseModel
from typing import Optional

client = OpenAI()

# Define the structured output schema
class InvoiceItem(BaseModel):
    description: str
    quantity: Optional[float] = None
    unit_price: Optional[float] = None
    total: float

class Invoice(BaseModel):
    vendor_name: str
    invoice_number: Optional[str] = None
    invoice_date: Optional[str] = None
    line_items: list[InvoiceItem]
    subtotal: Optional[float] = None
    tax: Optional[float] = None
    total_amount: float
    currency: str = "USD"

def extract_invoice_data(image_path: str) -> Invoice:
    """
    Complete pipeline: image → VLM extraction → structured Invoice object.
    Uses GPT-4o with structured outputs for reliable JSON extraction.
    """
    # Step 1: Load and encode the image
    with open(image_path, "rb") as f:
        image_b64 = base64.standard_b64encode(f.read()).decode("utf-8")

    ext = Path(image_path).suffix.lower().lstrip(".")
    mime = {"jpg": "image/jpeg", "jpeg": "image/jpeg",
            "png": "image/png", "pdf": "application/pdf"}.get(ext, "image/jpeg")

    # Step 2: Send to VLM with structured output schema
    response = client.beta.chat.completions.parse(
        model="gpt-4o-2024-08-06",
        messages=[
            {
                "role": "system",
                "content": (
                    "You are a precise document extraction assistant. "
                    "Extract all invoice data from the provided image. "
                    "Be exact — do not guess values you can't clearly see."
                )
            },
            {
                "role": "user",
                "content": [
                    {
                        "type": "image_url",
                        "image_url": {
                            "url": f"data:{mime};base64,{image_b64}",
                            "detail": "high"
                        }
                    },
                    {
                        "type": "text",
                        "text": "Extract all invoice data from this document."
                    }
                ]
            }
        ],
        response_format=Invoice,
        max_tokens=2048
    )

    invoice = response.choices[0].message.parsed

    # Step 3: Generate a human-readable report
    report = generate_invoice_report(invoice)
    print(report)

    return invoice

def generate_invoice_report(invoice: Invoice) -> str:
    """Convert structured Invoice data to a formatted text report."""
    lines = [
        f"INVOICE REPORT",
        f"=" * 40,
        f"Vendor:       {invoice.vendor_name}",
        f"Invoice No:   {invoice.invoice_number or 'N/A'}",
        f"Date:         {invoice.invoice_date or 'N/A'}",
        f"",
        f"LINE ITEMS:",
    ]
    for item in invoice.line_items:
        qty = f"{item.quantity} × " if item.quantity else ""
        price = f"${item.unit_price:.2f} " if item.unit_price else ""
        lines.append(f"  • {item.description}: {qty}{price}= ${item.total:.2f}")

    lines.extend([
        f"",
        f"Subtotal:     ${invoice.subtotal:.2f}" if invoice.subtotal else "",
        f"Tax:          ${invoice.tax:.2f}" if invoice.tax else "",
        f"TOTAL:        ${invoice.total_amount:.2f} {invoice.currency}",
    ])
    return "\n".join(l for l in lines if l is not None)

# Run the pipeline
invoice = extract_invoice_data("invoice_scan.jpg")
print(json.dumps(invoice.model_dump(), indent=2))

Handling Multiple Images

VLMs can receive multiple images in a single request — useful for comparing before/after states, analyzing a multi-page document, or reviewing a sequence of screenshots. Just add multiple image_url content items to the user message. Be mindful of token budget: each high-resolution image can consume 500–2000 tokens.

Performance Tip

Use "detail": "low" for large batches of images where you need rough analysis (e.g., classifying many photos). Use "detail": "high" only when you need to read small text, fine details, or analyze dense charts. Low detail is ~4× cheaper and faster.

Lab: Image Report Generator
Build an end-to-end pipeline: image upload → vision analysis → structured data → text report

In this lab you will build a complete multimodal pipeline that accepts an image (receipt, chart, or form), extracts structured data using a Vision Language Model, and generates a polished text report.

  1. Choose your image type — Pick one of: a grocery receipt, a bar chart from a slide deck, or a simple form. Download or photograph a sample image.
  2. Set up the vision call — Load the image, base64-encode it, and make your first GPT-4o or Claude 3 API call asking "What do you see in this image?" to verify the pipeline works.
  3. Define a structured output schema — Use Pydantic to define the data model appropriate for your image type (e.g., Receipt with items: list[LineItem] and total: float). Use GPT-4o structured outputs to enforce the schema.
  4. Extract the data — Update your API call to return the structured output. Print the raw JSON to verify correctness.
  5. Generate the report — Write a function that converts your Pydantic model to a human-readable text report. For a receipt: itemized list + total. For a chart: key insights + data points. For a form: field-value pairs.
  6. Test edge cases — Try a low-quality or partially obscured image. What does the model return for fields it can't read? Update your schema to handle optional fields gracefully.
  7. Bonus: Add audio output — Pass your generated report text to the OpenAI TTS API and save it as an MP3. You've now built a full image → text → audio pipeline.

Training vs Inference

Every ML system has two distinct phases. Confusing them leads to fundamental design mistakes.

🏋️

Training

The model sees (features, label) pairs and adjusts its internal weights to minimize prediction errors. Happens once (or periodically). Can take hours to months. Uses massive compute.

Analogy: A student studying for an exam — reading, practicing, correcting mistakes.

⚡

Inference

The trained model receives new input and produces a prediction. No weight updates. Happens millions of times per day in production. Must be fast.

Analogy: The same student taking the exam — applying what they learned to new questions.

💡

Why This Matters for Engineering

Training infrastructure (GPUs, large datasets, long jobs) is completely different from inference infrastructure (low latency, high throughput, caching). Most of your production ML work is optimizing inference.

Loss Functions and Gradient Descent

Training is fundamentally an optimization problem: find the weights that minimize prediction error. The mechanism is gradient descent.

  1. Forward pass: Feed input through the model, get a prediction.
  2. Calculate loss: Measure how wrong the prediction is (e.g., Mean Squared Error for regression).
  3. Backward pass: Calculate gradients — how much each weight contributed to the error.
  4. Update weights: Nudge each weight in the direction that reduces loss (learning rate × gradient).
  5. Repeat over all training examples (one epoch). Repeat for many epochs.
import numpy as np

# Simple gradient descent for linear regression: y = w*x + b
# Goal: find w and b that minimize MSE on training data

X = np.array([1, 2, 3, 4, 5])      # house size (hundreds of sqft)
y = np.array([150, 200, 250, 300, 350])  # price (thousands)

w, b = 0.0, 0.0  # Initialize weights randomly
lr = 0.01        # Learning rate
epochs = 1000

for epoch in range(epochs):
    y_pred = w * X + b          # Forward pass
    loss = np.mean((y_pred - y) ** 2)   # MSE loss

    # Gradients (calculus: partial derivatives of loss w.r.t. w and b)
    dw = np.mean(2 * (y_pred - y) * X)
    db = np.mean(2 * (y_pred - y))

    w -= lr * dw   # Update weights
    b -= lr * db

    if epoch % 200 == 0:
        print(f"Epoch {epoch}: loss={loss:.2f}, w={w:.2f}, b={b:.2f}")

print(f"\nFinal: y = {w:.1f}x + {b:.1f}")
# Predict: house with 3.5 (350 sqft) → price?
print(f"Prediction for 350 sqft: ${w*3.5 + b:.0f}k")

Regression vs Classification

TypeOutputExamplesCommon Algorithms
RegressionA continuous numberHouse price, temperature, stock price, customer lifetime valueLinear regression, Random Forest, XGBoost, Neural nets
ClassificationA category (class)Spam/not spam, cat/dog, sentiment (pos/neg/neutral), disease diagnosisLogistic regression, SVM, Random Forest, Neural nets
 
from sklearn.linear_model import LinearRegression, LogisticRegression
from sklearn.datasets import make_regression, make_classification
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, accuracy_score

# REGRESSION: predict house price
X_reg, y_reg = make_regression(n_samples=200, n_features=5, noise=20, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X_reg, y_reg, test_size=0.2)

reg_model = LinearRegression()
reg_model.fit(X_train, y_train)
y_pred = reg_model.predict(X_test)
print(f"Regression MSE: {mean_squared_error(y_test, y_pred):.2f}")

# CLASSIFICATION: spam detection (binary)
X_clf, y_clf = make_classification(n_samples=200, n_features=10, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X_clf, y_clf, test_size=0.2)

clf_model = LogisticRegression()
clf_model.fit(X_train, y_train)
y_pred = clf_model.predict(X_test)
print(f"Classification Accuracy: {accuracy_score(y_test, y_pred):.2%}")

Overfitting vs Underfitting

📉

Underfitting

Model is too simple to capture the patterns. High training error AND high test error. Fix: more complex model, more features, less regularization.

✅

Good Fit

Low training error AND low test error. The model generalizes. This is the goal.

📈

Overfitting

Model memorized training data. Low training error, HIGH test error. Fix: more data, dropout, regularization, simpler model.

Quick Check

❓ A model has 99% training accuracy but 60% test accuracy. What is happening?▼
Overfitting. The model memorized training examples instead of learning general patterns. The 39-point gap between train and test accuracy is the telltale sign.
❓ Predict whether a customer will churn (leave) vs predict how much they'll spend next month. Which task type is each?▼
Churn prediction = Classification (binary: churn / not churn). Spend prediction = Regression (continuous dollar amount).

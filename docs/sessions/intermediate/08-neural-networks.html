The Artificial Neuron

A biological neuron fires when the sum of its incoming signals exceeds a threshold. An artificial neuron mimics this: it takes a weighted sum of its inputs, adds a bias term, and passes the result through an activation function to produce an output.

The math: output = activation(w₁x₁ + w₂x₂ + w₃x₃ + b)

The weights w control how much each input matters. The bias b shifts the activation threshold. Training adjusts both to minimize loss.

Common activation functions and when to use them:

Activation Output Range Use Case Watch Out For
ReLU [0, ∞) Default choice for hidden layers in most networks Dying ReLU: neurons that always output 0
Sigmoid (0, 1) Binary classification output layer Vanishing gradients in deep networks
Tanh (-1, 1) Hidden layers in RNNs Vanishing gradients (less severe than sigmoid)
Softmax (0, 1) summing to 1 Multi-class classification output layer Only meaningful as output, not hidden layer

Layers: Input, Hidden, Output

Neurons are organized into layers. Every layer takes the previous layer's outputs as its inputs, and feeds its outputs to the next layer. This is a feedforward network because data flows in one direction.

The number of hidden layers is the network's depth — this is what "deep" in deep learning refers to. Deeper networks can represent more abstract and complex patterns, but require more data and are harder to train.

Each layer learns a progressively more abstract representation of the input. In a face recognition network: layer 1 detects edges → layer 2 detects shapes → layer 3 detects facial features → output predicts identity.

Loss Functions & Backpropagation

After a forward pass, the network has made predictions. The loss function measures how wrong those predictions are. During training, we want to minimize this loss by finding the weights that produce the best predictions.

The intuition for gradient descent is a ball rolling downhill. The loss function defines a landscape of hills and valleys. Gradient descent follows the slope (gradient) downward, nudging weights in the direction that reduces loss.

Backpropagation is the algorithm that efficiently computes these gradients. It applies the chain rule from calculus, working backwards from the loss through each layer, computing how much each weight contributed to the error.

Core Insight: Backpropagation is just the chain rule from calculus applied repeatedly through a composition of functions. It's a clever bookkeeping algorithm, not magic. Modern frameworks (PyTorch, TensorFlow) compute this automatically via automatic differentiation.

Training a Neural Network

Three hyperparameters dominate neural network training:

  • Epochs: How many times to pass through the entire training dataset. More epochs = more learning, but eventually diminishing returns (and overfitting).
  • Batch size: How many samples to process before updating weights. Small batches (32–128) are noisier but more regularizing. Large batches (512+) converge more smoothly but need more memory.
  • Learning rate: How large each weight update step is. Too large = overshoots the minimum. Too small = takes forever. Typical starting value: 1e-3.
mnist_pytorch.py 
import torch
import torch.nn as nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# 1. Load MNIST handwritten digits
transform = transforms.Compose([transforms.ToTensor(), transforms.Normalize((0.5,), (0.5,))])
train_data = datasets.MNIST("data/", train=True, download=True, transform=transform)
test_data  = datasets.MNIST("data/", train=False, transform=transform)
train_loader = DataLoader(train_data, batch_size=64, shuffle=True)
test_loader  = DataLoader(test_data, batch_size=64)

# 2. Define the network (3 fully connected layers)
class DigitClassifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Flatten(),              # 28x28 image → 784-dim vector
            nn.Linear(784, 256),
            nn.ReLU(),
            nn.Dropout(0.3),           # regularization: randomly zero 30% of neurons
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, 10)         # 10 output classes (digits 0-9)
        )
    def forward(self, x):
        return self.net(x)

model = DigitClassifier()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.CrossEntropyLoss()

# 3. Training loop
for epoch in range(10):
    model.train()
    running_loss = 0
    for images, labels in train_loader:
        optimizer.zero_grad()              # clear old gradients
        output = model(images)             # forward pass
        loss = criterion(output, labels)   # compute loss
        loss.backward()                    # backpropagation
        optimizer.step()                   # update weights
        running_loss += loss.item()

    # 4. Evaluate on test set each epoch
    model.eval()
    correct = 0
    with torch.no_grad():
        for images, labels in test_loader:
            preds = model(images).argmax(dim=1)
            correct += (preds == labels).sum().item()

    acc = correct / len(test_data)
    print(f"Epoch {epoch+1:2d} | loss: {running_loss/len(train_loader):.3f} | test acc: {acc:.3f}")

Common Network Types

Different architectures have been developed to exploit the structure of different data types. Using the right architecture is one of the most impactful design decisions.

Architecture Key Idea Best For Famous Examples
MLP
(Fully Connected)		 Every neuron connected to every neuron in next layer Tabular data, simple classification Early neural nets, regression problems
CNN
(Convolutional)		 Learnable filters detect local spatial patterns; weight sharing Images, video, spatial data ResNet, EfficientNet, YOLO
RNN / LSTM
(Recurrent)		 Maintains hidden state across sequence steps; handles variable length Time series, speech, older NLP LSTM, GRU, seq2seq translation
Transformer
(Attention-based)		 Self-attention lets every token attend to every other token; parallelizable Language, code, images (ViT), multimodal GPT-4, BERT, Claude, LLaMA

    import matplotlib.pyplot as plt

model.eval()
mistakes_x, mistakes_true, mistakes_pred = [], [], []

with torch.no_grad():
    for images, labels in test_loader:
        preds = model(images).argmax(dim=1)
        wrong = (preds != labels)
        mistakes_x.extend(images[wrong])
        mistakes_true.extend(labels[wrong])
        mistakes_pred.extend(preds[wrong])
        if len(mistakes_x) >= 16:
            break

Embeddings & Semantic Search

Intermediate 55 min Session 10 of 20

What Are Embeddings?

An embedding is a dense numerical representation of text — a list of floating-point numbers that encodes semantic meaning. When you pass a sentence to an embedding model, you get back a fixed-size vector: for example, OpenAI's text-embedding-3-small returns an array of 1536 floats for any input, whether it's one word or five paragraphs.

What makes embeddings powerful is that similar meanings produce similar vectors. The words "dog" and "puppy" will have vectors that are close together in this high-dimensional space. "Bank (financial institution)" and "bank (river bank)" will be farther apart. This spatial relationship is what makes semantic tasks possible.

Text Embedding (simplified 4-D) Relationship
"The car engine failed" [0.82, 0.14, 0.61, 0.33] Very similar
cosine ≈ 0.96
"The automobile motor broke down" [0.79, 0.18, 0.58, 0.37]
"I love pasta carbonara" [0.12, 0.91, 0.08, 0.74] Unrelated
cosine ≈ 0.11

The dimensionality (1536, 3072, 768, etc.) represents the model's capacity to capture nuanced meaning. More dimensions generally mean finer-grained semantic distinctions, at the cost of storage and compute.

Core Concept

Embeddings turn text into numbers that preserve semantic meaning. The closer two vectors are in this space, the more similar their meanings — regardless of the exact words used.

Cosine Similarity

Cosine similarity is the standard way to measure how similar two embedding vectors are. Rather than measuring the straight-line distance between them (Euclidean distance), it measures the angle between the two vectors. This makes it robust to differences in vector magnitude — a short text and a long text covering the same topic will have a small angle, even though their raw vector lengths differ.

The formula is:

cosine_similarity(A, B) = (A · B) / (||A|| × ||B||)

The dot product (A · B) sums the element-wise products. The denominator normalizes by each vector's magnitude. The result is always between -1 and 1: 1 means identical direction, 0 means perpendicular (unrelated), -1 means opposite.

cosine_similarity.py
import numpy as np

def cosine_similarity(a, b):
    """Compute cosine similarity between two 1-D vectors."""
    dot_product = np.dot(a, b)
    magnitude_a = np.linalg.norm(a)
    magnitude_b = np.linalg.norm(b)
    return dot_product / (magnitude_a * magnitude_b)

# Example: two related sentences (simplified 4-D vectors)
car_engine    = np.array([0.82, 0.14, 0.61, 0.33])
auto_motor    = np.array([0.79, 0.18, 0.58, 0.37])
pasta_recipe  = np.array([0.12, 0.91, 0.08, 0.74])

print(f"car_engine vs auto_motor:   {cosine_similarity(car_engine, auto_motor):.4f}")
# Output: ~0.9987 — very similar
print(f"car_engine vs pasta_recipe: {cosine_similarity(car_engine, pasta_recipe):.4f}")
# Output: ~0.3142 — unrelated

# With numpy broadcasting for batch similarity
from sklearn.metrics.pairwise import cosine_similarity as sk_cosine

vectors = np.stack([car_engine, auto_motor, pasta_recipe])
query   = car_engine.reshape(1, -1)

scores = sk_cosine(query, vectors)[0]
ranked = sorted(enumerate(scores), key=lambda x: x[1], reverse=True)
for idx, score in ranked:
    print(f"  [{score:.4f}] index {idx}")

Semantic Search vs Keyword Search

Traditional search engines match exact words. If a user searches "car problems", a keyword search won't find a document that only uses the word "automobile" — even though the meaning is identical. Semantic search bridges this gap by comparing meaning instead of characters.

Property Keyword Search Semantic Search
Matching method Exact word / token match (TF-IDF, BM25) Vector similarity in embedding space
Synonyms Misses unless explicitly added Handled naturally
Paraphrases Fails ("I need a vehicle" vs "car") Works well
Typos & abbreviations Fails without fuzzy matching Often robust
Precision for exact terms Excellent (product codes, names) Can be less precise
Computational cost Low — inverted index Higher — ANN search over vectors

Query: "How do I fix my car?" — Semantic search also retrieves:

  • "Automobile repair guide" — synonym match
  • "Vehicle maintenance tips" — conceptual match
  • "My engine won't start" — contextual match
Hybrid Search

Production search systems often combine both: BM25 for keyword precision plus embeddings for semantic recall. Pinecone, Weaviate, and Elasticsearch all support hybrid search modes.

Building a Semantic Search System

A semantic search system has two distinct phases: offline indexing (run once) and online retrieval (run per query). The offline phase is expensive; the online phase should be fast.

Four steps to build it:

  1. Embed documents: Pass each document (or chunk) through an embedding model to get a vector.
  2. Store in a vector database: Persist vectors with their metadata for fast similarity lookup.
  3. Embed the query: At query time, embed the user's question with the same model.
  4. Find nearest neighbors: Retrieve the top-k most similar document vectors using ANN (approximate nearest neighbor) search.

Popular vector databases:

Database Type Best For
Pinecone Managed cloud Production, serverless, no infrastructure
Weaviate Open source / cloud Hybrid search, GraphQL API
Chroma Open source (local-first) Prototyping, local dev, Python-native
pgvector Postgres extension Teams already using Postgres
semantic_search.py
from sentence_transformers import SentenceTransformer
import chromadb
import numpy as np

# --- 1. Load an open-source embedding model (free, runs locally) ---
model = SentenceTransformer("all-MiniLM-L6-v2")  # 384-dimensional vectors

# --- 2. Your document corpus ---
documents = [
    "How to change a car tire",
    "Automobile engine repair guide",
    "Python machine learning tutorial",
    "Deep learning with PyTorch",
    "Cooking pasta carbonara recipe",
    "Vehicle brake maintenance tips",
]
doc_ids = [f"doc_{i}" for i in range(len(documents))]

# --- 3. Embed documents ---
doc_embeddings = model.encode(documents, normalize_embeddings=True)
print(f"Embedding shape: {doc_embeddings.shape}")  # (6, 384)

# --- 4. Store in Chroma (in-memory for demo) ---
client = chromadb.Client()
collection = client.create_collection("course_docs")
collection.add(
    documents=documents,
    embeddings=doc_embeddings.tolist(),
    ids=doc_ids
)

# --- 5. Search at query time ---
query = "My vehicle needs repair"
query_embedding = model.encode([query], normalize_embeddings=True)

results = collection.query(
    query_embeddings=query_embedding.tolist(),
    n_results=3
)

print(f"\nQuery: '{query}'")
print("Top 3 results:")
for i, (doc, distance) in enumerate(
    zip(results["documents"][0], results["distances"][0])
):
    print(f"  {i+1}. [{1 - distance:.3f}] {doc}")

Applications of Embeddings

Embeddings are a general-purpose representation layer that unlocks many tasks beyond just search. Here are three common production use cases with code.

1. Similarity Scoring

Score how similar two pieces of text are — useful for review deduplication, plagiarism detection, or FAQ matching.

similarity_scoring.py
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer("all-MiniLM-L6-v2")

ticket_1 = "App crashes when I tap the login button"
ticket_2 = "The app keeps closing on the sign-in screen"
ticket_3 = "How do I cancel my subscription?"

embs = model.encode([ticket_1, ticket_2, ticket_3])

sim_1_2 = util.cos_sim(embs[0], embs[1]).item()
sim_1_3 = util.cos_sim(embs[0], embs[2]).item()

print(f"Ticket 1 vs 2 (likely duplicate): {sim_1_2:.3f}")  # ~0.89
print(f"Ticket 1 vs 3 (different issue):  {sim_1_3:.3f}")  # ~0.21

2. Clustering Topics

Group documents into topics automatically — no labels needed. Useful for organizing a knowledge base or discovering themes in customer feedback.

clustering.py
from sentence_transformers import SentenceTransformer
from sklearn.cluster import KMeans

model = SentenceTransformer("all-MiniLM-L6-v2")

feedback = [
    "The checkout is too slow",
    "Payment keeps failing",
    "I love the new dark mode",
    "The UI looks great on mobile",
    "Order confirmation emails are delayed",
    "Billing didn't go through",
]

embeddings = model.encode(feedback)
kmeans = KMeans(n_clusters=3, random_state=42)
labels = kmeans.fit_predict(embeddings)

for text, cluster in zip(feedback, labels):
    print(f"  Cluster {cluster}: {text}")

3. Recommendation System

Find content similar to what a user just viewed or liked — embeddings power the "more like this" feature.

recommendations.py
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer("all-MiniLM-L6-v2")

articles = [
    "Introduction to neural networks",
    "Deep learning for computer vision",
    "Transformer architecture explained",
    "Python for beginners",
    "How to bake sourdough bread",
    "Garden vegetable growing guide",
]

embeddings = model.encode(articles, normalize_embeddings=True)

# User just read article 0 — find similar ones
viewed = embeddings[0]
scores = util.cos_sim(viewed, embeddings)[0]

# Sort by similarity, skip the viewed article itself
ranked = sorted(
    [(i, s.item()) for i, s in enumerate(scores) if i != 0],
    key=lambda x: x[1], reverse=True
)
print("Recommended (based on article 0):")
for idx, score in ranked[:3]:
    print(f"  [{score:.3f}] {articles[idx]}")

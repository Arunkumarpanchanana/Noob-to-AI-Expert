AI App Architecture Patterns

Before writing a single line of code, the most important question is: which pattern fits your problem? The right choice determines your latency, cost, complexity, and maintenance burden. Here are the four main patterns and when to use each.

Pattern Complexity When to Use Example
Simple LLM Call Low Single-turn tasks; the model's training knowledge is sufficient Writing assistant, code explainer, grammar checker
RAG Medium Private or up-to-date knowledge required; facts must be verifiable Internal docs chatbot, support bot over your knowledge base
Agent High Multi-step tasks; model needs to use tools, make decisions, iterate Research assistant, code execution bot, autonomous workflow
Fine-tuned Model Very High Consistent style/format needed; task is very domain-specific; prompt alone insufficient Legal doc drafting in your firm's style, specialized classifier
Start Simple

Always start with the simplest pattern that could work. A well-engineered system prompt + simple LLM call solves 80% of use cases. Only add RAG, agents, or fine-tuning when you've validated that the simpler approach isn't sufficient.

Building a Backend API

A FastAPI backend is the standard pattern for exposing an LLM to a frontend. FastAPI is chosen for AI apps because it's async-native (critical for streaming), has automatic OpenAPI docs, and its type annotations make prompt engineering more structured.

main.py
import os
from fastapi import FastAPI, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel
from openai import AsyncOpenAI
from fastapi.responses import StreamingResponse
import json

app = FastAPI(title="AI FAQ Bot API")

# Allow your frontend origin (adjust in production)
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "https://yourdomain.com"],
    allow_methods=["POST"],
    allow_headers=["*"],
)

client = AsyncOpenAI(api_key=os.environ["OPENAI_API_KEY"])

class ChatRequest(BaseModel):
    question: str
    history: list[dict] = []   # [{"role": "user"/"assistant", "content": "..."}]

SYSTEM_PROMPT = """You are a helpful AI assistant. Be concise and accurate.
If you don't know something, say so rather than guessing."""

@app.post("/chat")
async def chat(req: ChatRequest):
    """Answer a question, with optional conversation history."""
    if not req.question.strip():
        raise HTTPException(status_code=400, detail="Question cannot be empty")

    messages = [{"role": "system", "content": SYSTEM_PROMPT}]
    messages.extend(req.history[-10:])   # keep last 10 turns for context
    messages.append({"role": "user", "content": req.question})

    response = await client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
    )
    return {"answer": response.choices[0].message.content}

@app.post("/chat/stream")
async def chat_stream(req: ChatRequest):
    """Streaming version — returns Server-Sent Events."""
    messages = [{"role": "system", "content": SYSTEM_PROMPT}]
    messages.extend(req.history[-10:])
    messages.append({"role": "user", "content": req.question})

    async def event_generator():
        async with client.chat.completions.stream(
            model="gpt-4o-mini",
            messages=messages,
        ) as stream:
            async for chunk in stream:
                delta = chunk.choices[0].delta.content
                if delta:
                    yield f"data: {json.dumps({'token': delta})}\n\n"
        yield "data: [DONE]\n\n"

    return StreamingResponse(event_generator(), media_type="text/event-stream")

# Run with: uvicorn main:app --reload

Frontend Integration

A minimal chat UI only needs vanilla JavaScript — no React required. For streaming, the browser's EventSource API (for GET) or a manual fetch with ReadableStream (for POST) handles SSE from your FastAPI backend.

index.html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>AI Chat</title>
  <style>
    body { font-family: sans-serif; max-width: 700px; margin: 2rem auto; padding: 0 1rem; }

                overflow-y: auto; padding: 1rem; margin-bottom: 1rem; }
    .user   { text-align: right; color: #333; margin: 0.5rem 0; }
    .bot    { text-align: left;  color: #6c63ff; margin: 0.5rem 0; }
    #form   { display: flex; gap: 0.5rem; }
    #input  { flex: 1; padding: 0.5rem; border: 1px solid #ddd; border-radius: 6px; }
    button  { padding: 0.5rem 1rem; background: #6c63ff; color: #fff;
              border: none; border-radius: 6px; cursor: pointer; }
  </style>
</head>
<body>
  <h2>AI Chat</h2>
  <div id="messages"></div>
  <form id="form">
    <input id="input" placeholder="Ask something..." autocomplete="off">
    <button type="submit">Send</button>
  </form>

  <script>
    const API = "http://localhost:8000";
    const history = [];

    document.getElementById("form").addEventListener("submit", async (e) => {
      e.preventDefault();
      const question = document.getElementById("input").value.trim();
      if (!question) return;
      document.getElementById("input").value = "";

      appendMessage("user", question);
      history.push({ role: "user", content: question });

      // Streaming fetch via ReadableStream
      const botDiv = appendMessage("bot", "");
      const res = await fetch(`${API}/chat/stream`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ question, history: history.slice(0, -1) }),
      });

      const reader = res.body.getReader();
      const decoder = new TextDecoder();
      let fullReply = "";

      while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        const lines = decoder.decode(value).split("\n");
        for (const line of lines) {
          if (!line.startsWith("data: ")) continue;
          const payload = line.slice(6);
          if (payload === "[DONE]") break;
          const { token } = JSON.parse(payload);
          fullReply += token;
          botDiv.textContent = fullReply;
        }
      }
      history.push({ role: "assistant", content: fullReply });
    });

    function appendMessage(role, text) {
      const div = document.createElement("div");
      div.className = role;
      div.textContent = text;
      document.getElementById("messages").appendChild(div);
      div.scrollIntoView({ behavior: "smooth" });
      return div;
    }
  </script>
</body>
</html>

State Management

Conversation state — the history of messages — must be preserved across requests. There are two main approaches depending on your scale:

Approach Storage Pros Cons
Client-side Browser memory / localStorage Simple, no server state, works for single-user Lost on page refresh (unless stored in localStorage); client sends full history every request
Server-side (dict) In-memory Python dict, keyed by session_id Fast, no DB dependency Lost on server restart; doesn't scale horizontally
Server-side (DB) Redis, PostgreSQL, DynamoDB Persistent, scalable, multi-server More complex setup; latency per request
Stateless APIs

Stateless APIs need the client to send conversation history every time. If the client forgets a message or the history grows unboundedly, you'll hit context window limits. Always cap how much history you send — the last 10 turns is usually sufficient.

Deployment Basics

Getting your AI app into production doesn't require a DevOps team. Modern PaaS (Platform-as-a-Service) platforms handle servers, SSL, and deployments automatically.

Platform Best For Cost Notes
Railway FastAPI backends Free tier + usage-based Deploy from GitHub in 2 clicks; supports env vars
Render FastAPI / Docker Free tier; $7/mo for always-on Spins down on free tier after inactivity
Vercel Frontend + serverless functions Free for hobby Excellent for Next.js; not ideal for long-running streams
GitHub Pages Static frontends only Free No server code; perfect for this course's site

Environment variables in production: Never hardcode API keys. Every platform above provides a secrets/env vars UI. In Python, read them with os.environ["OPENAI_API_KEY"]. Use a .env file locally with python-dotenv — but never commit it.

Static Frontend Strategy

For a static frontend with LLM API calls, deploy to GitHub Pages and use the LLM API directly from the browser — but keep your API key server-side in any production app. For hobbyist/demo projects, a lightweight serverless function (e.g., Vercel Edge Function) that forwards requests to the LLM API is the cleanest approach.

Lab: Build and Deploy a FAQ Bot
Create a FastAPI backend, HTML frontend, test locally, and deploy

You'll build a complete AI FAQ bot from scratch — backend, frontend, and a live deployment. By the end you'll have a public URL you can share.

  1. Set up the project
    Create a new directory with requirements.txt containing fastapi uvicorn openai python-dotenv. Create a .env file with your OpenAI API key.
  2. Build the FastAPI backend
    Copy the main.py from this session. Add a custom SYSTEM_PROMPT for a topic of your choice (e.g., "You are a cooking expert assistant").
  3. Build the frontend
    Create an index.html using the streaming chat UI from this session. Style it to match your bot's theme.
  4. Test locally
    Run uvicorn main:app --reload. Open index.html in a browser (use Live Server or similar). Test 5 questions. Check that conversation history is maintained.
  5. Deploy to Railway
    Push to a GitHub repo. Connect Railway to the repo. Add the OPENAI_API_KEY environment variable in Railway's settings. Deploy and get your public URL.

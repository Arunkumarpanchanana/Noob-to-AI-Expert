AI App Architecture

A production AI app has three tiers. Understanding this architecture lets you build, debug, and scale any AI product.

🖥️

Frontend

What users see. Streamlit for rapid prototypes. React/Next.js for production. Handles input, displays output.

⚙️

Backend API

FastAPI or Flask. Receives requests, calls LLM API, applies business logic, returns structured responses.

🤖

AI Layer

LLM API calls, vector DB queries, embeddings. The intelligence. Sits behind the backend, never exposed directly to users.

FastAPI Backend

# pip install fastapi uvicorn anthropic python-dotenv
# Run: uvicorn main:app --reload

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import anthropic
import os

app = FastAPI(title="AI FAQ Bot API")
client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

SYSTEM_PROMPT = """You are a helpful FAQ assistant. Answer concisely.
If you don't know the answer, say so clearly."""

class ChatRequest(BaseModel):
    message: str
    history: list[dict] = []

class ChatResponse(BaseModel):
    reply: str
    tokens_used: int

@app.post("/chat", response_model=ChatResponse)
async def chat(request: ChatRequest):
    if not request.message.strip():
        raise HTTPException(status_code=400, detail="Empty message")

    messages = request.history + [{"role": "user", "content": request.message}]

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=512,
        system=SYSTEM_PROMPT,
        messages=messages
    )

    return ChatResponse(
        reply=response.content[0].text,
        tokens_used=response.usage.input_tokens + response.usage.output_tokens
    )

@app.get("/health")
async def health(): return {"status": "ok"}

Streamlit Frontend

# pip install streamlit requests
# Run: streamlit run app.py

import streamlit as st
import requests

st.set_page_config(page_title="AI FAQ Bot", page_icon="🤖", layout="centered")
st.title("🤖 AI FAQ Bot")
st.caption("Ask me anything!")

# Initialize session state
if "messages" not in st.session_state:
    st.session_state.messages = []

# Display conversation history
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.write(msg["content"])

# Chat input
if prompt := st.chat_input("Type your question..."):
    # Display user message
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.write(prompt)

    # Get AI response
    with st.chat_message("assistant"):
        with st.spinner("Thinking..."):
            try:
                response = requests.post(
                    "http://localhost:8000/chat",
                    json={"message": prompt, "history": st.session_state.messages[:-1]},
                    timeout=30
                )
                reply = response.json()["reply"]
                tokens = response.json()["tokens_used"]
            except Exception as e:
                reply = f"Error: {e}"
                tokens = 0

        st.write(reply)
        st.caption(f"Tokens used: {tokens}")

    st.session_state.messages.append({"role": "assistant", "content": reply})

Fastest Path to Deployed

For a Streamlit app: push to GitHub → go to share.streamlit.io → select repo → add ANTHROPIC_API_KEY as secret → deploy. Live in under 5 minutes. No servers, no DevOps.

Lab: Deploy an AI FAQ Bot to the Internet

Estimated time: 60 minutes · Your local machine + Streamlit Cloud

Colab for Code Streamlit Cloud
1

Build the Streamlit App Locally

Copy the Streamlit code from the Learn tab. Run it locally with streamlit run app.py. Chat with it. It should work against your local backend or directly with the API.

2

Add Your Domain Knowledge

Customize the system prompt to make it an expert on something you know: a game, a technology, a subject area. Add 5 "FAQ" items hardcoded in the system prompt.

3

Push to GitHub and Deploy

Create a new GitHub repo. Push your app.py and requirements.txt. Go to share.streamlit.io, connect the repo, add your API key as a secret, and deploy.

4

Share the Link

You now have a live AI app on the internet. Share the URL. This is your first shipped AI product!

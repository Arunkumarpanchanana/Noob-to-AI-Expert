LLM Evaluation

You can't improve what you can't measure. Build golden datasets, automated eval suites, and LLM-as-judge pipelines to systematically improve your AI apps.

75 min Expert Colab Lab 5 Quiz Questions

Why Evaluation Matters

Without evals, you're flying blind. You can't safely upgrade models, change prompts, or add features without knowing if you've regressed. Evals are to AI apps what tests are to software.

👤

Human Eval

Highest quality. Experts rate outputs on rubrics. Slow and expensive. Use for: final validation, edge case review.

🤖

LLM-as-Judge

Strong LLM grades outputs. Scales to thousands of examples. 70–90% correlation with human judges.

⚙️

Automated

Code-based checks: exact match, regex, JSON validity, substring presence. Fast, cheap, 100% reproducible.

Building a Golden Dataset

A golden dataset is a set of (input, expected output) pairs where humans have verified the correct answer. It's the foundation of all systematic evaluation.

import json
from dataclasses import dataclass, asdict
from typing import Literal

@dataclass
class EvalCase:
    id: str
    input: str
    expected_output: str
    expected_intent: str
    difficulty: Literal["easy", "medium", "hard"]
    tags: list[str]

golden_dataset = [
    EvalCase(
        id="cs_001",
        input="How do I reset my password?",
        expected_output="Go to login page → click 'Forgot password' → enter email → check inbox",
        expected_intent="how_to",
        difficulty="easy",
        tags=["authentication", "self_serve"]
    ),
    EvalCase(
        id="cs_002",
        input="I was charged twice last month",
        expected_output="I'm sorry about that. I'm escalating this to our billing team who will contact you within 24 hours.",
        expected_intent="billing_dispute",
        difficulty="medium",
        tags=["billing", "escalation"]
    ),
    EvalCase(
        id="cs_003",
        input="Can you tell me your competitors' pricing?",
        expected_output="I'm not able to discuss competitor pricing. I'd be happy to walk you through our pricing plans.",
        expected_intent="competitor_question",
        difficulty="hard",
        tags=["guardrail", "competitor"]
    ),
]

# Save dataset
with open("golden_dataset.json", "w") as f:
    json.dump([asdict(c) for c in golden_dataset], f, indent=2)

print(f"Saved {len(golden_dataset)} eval cases")

LLM-as-Judge

Use a strong model to evaluate outputs from another model. This scales to thousands of test cases while maintaining quality close to human judgment.

import anthropic

client = anthropic.Anthropic()

JUDGE_SYSTEM = """You are an expert evaluator for AI customer support responses.
Score responses on a 1-5 scale on these dimensions:
- accuracy: Is the information correct?
- helpfulness: Does it solve the user's problem?
- tone: Is it professional and empathetic?
- safety: Does it avoid violating any constraints (like discussing competitors)?

Return ONLY valid JSON: {"accuracy": N, "helpfulness": N, "tone": N, "safety": N, "rationale": "..."}"""

def llm_judge(user_input: str, model_output: str, expected: str) -> dict:
    prompt = f"""User question: {user_input}

Expected response (reference): {expected}

Actual model response: {model_output}

Evaluate the actual response."""

    response = client.messages.create(
        model="claude-opus-4-7",
        max_tokens=512,
        system=JUDGE_SYSTEM,
        messages=[{"role": "user", "content": prompt}]
    )

    import json, re
    text = response.content[0].text
    match = re.search(r'\{.*\}', text, re.DOTALL)
    if match:
        return json.loads(match.group())
    return {"error": "parse_failed", "raw": text}

# Run evaluation
result = llm_judge(
    user_input="How do I reset my password?",
    model_output="To reset your password: go to the login page and click 'Forgot password', then follow the email instructions.",
    expected="Go to login page → click 'Forgot password' → enter email → check inbox"
)
print(result)
# {"accuracy": 5, "helpfulness": 5, "tone": 4, "safety": 5, "rationale": "..."}

Regression Testing

Run your full eval suite every time you change a prompt, upgrade a model, or deploy a new version. A regression is when a score drops compared to baseline.

import statistics

def run_eval_suite(model_fn, dataset: list[EvalCase]) -> dict:
    """Run the full eval suite and return aggregate metrics."""
    results = []

    for case in dataset:
        output = model_fn(case.input)
        scores = llm_judge(case.input, output, case.expected_output)
        results.append({
            "case_id": case.id,
            "difficulty": case.difficulty,
            "scores": scores,
            "output": output
        })

    # Aggregate metrics
    all_accuracy = [r["scores"].get("accuracy", 0) for r in results]
    all_helpfulness = [r["scores"].get("helpfulness", 0) for r in results]

    return {
        "total_cases": len(results),
        "avg_accuracy": statistics.mean(all_accuracy),
        "avg_helpfulness": statistics.mean(all_helpfulness),
        "pass_rate": sum(1 for r in results if r["scores"].get("accuracy", 0) >= 4) / len(results),
        "results": results
    }

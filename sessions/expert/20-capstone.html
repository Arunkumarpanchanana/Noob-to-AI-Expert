What Makes a Great Capstone

A capstone project demonstrates mastery across the full stack: you understand the problem, design the architecture, implement it cleanly, evaluate it systematically, and deploy it to production. It becomes a portfolio piece.

🎯

Focused Scope

One problem, solved well. Better to build a polished FAQ bot than a half-finished "AI everything" platform.

📏

Evaluatable

You must have metrics. What does "good" mean for your app? Implement an eval suite before you call it done.

🚀

Deployed

It must be live. A working URL is worth more than beautiful code that runs only on your laptop.

Submission Checklist

✅ Pre-launch checklist — click to expand▼
  • GitHub repo is public with clear README
  • API key is stored as an environment variable, never in code
  • System prompt is well-crafted with persona, constraints, and format
  • At least one retrieval mechanism (RAG or tool calling)
  • 20+ golden test cases with automated eval
  • Response caching for repeated queries
  • Retry logic with exponential backoff
  • Per-user rate limiting
  • Request logging with tokens + cost
  • Live deployed URL tested from a fresh browser
  • Bias check: does it behave consistently across user groups?

Choose Your Project Track

Pick one. Build it well. Ship it.

🎓

Track A: AI Tutor

A subject-matter expert chatbot with a RAG backend over uploaded study materials. Explains concepts, quizzes the user, tracks what they've learned.

Key challenges: Pedagogy-aware system prompt · Knowledge graph for topic dependencies · Quiz generation from retrieved content

📄

Track B: Resume Reviewer

Users upload a resume and paste a job description. The app returns: match score, missing keywords, rewrite suggestions for each bullet point, and a one-click improved version.

Key challenges: PDF parsing · Structured JSON output · Diff view of original vs. improved · PII handling

📚

Track C: Knowledge Base Bot

Ingest a set of documents (company wiki, product docs, textbook). Users ask natural-language questions and get cited answers. Shows source snippets. Never hallucinates beyond the provided docs.

Key challenges: Chunking strategy · Citation tracking · "I don't know" detection · Hybrid search

🎤

Track D: Meeting Assistant

User uploads a meeting recording or transcript. App produces: structured summary, action items with owners, decisions made, follow-up questions, and a shareable HTML report.

Key challenges: Whisper transcription · Multi-speaker diarization · Action item extraction · HTML report generation

💻

Track E: AI Coding Assistant

A code review + generation tool. User pastes code or describes a function. App explains it, finds bugs, generates tests, and suggests improvements. Uses tool calling to run code in a sandbox.

Key challenges: Code execution sandbox · Test generation · Diff display · Language detection

1

Plan and Scaffold

Write a 1-page design doc: problem, users, architecture diagram, API endpoints, eval criteria. Create the repo and project structure. Don't write app logic yet — write tests first.

2

Build the Core Feature

Implement the minimum working version: the LLM call with a solid system prompt, the key retrieval or tool-calling logic, and a basic UI. Get it working end-to-end before polishing.

3

Build the Eval Suite

Create 25 golden test cases covering easy, medium, and hard scenarios. Implement LLM-as-judge evaluation. Run the full suite. Target ≥80% pass rate before deploying.

4

Production Harden and Deploy

Add caching, retries, rate limiting, logging, and PII protection. Deploy to Streamlit Cloud or Render. Share your live URL. Congratulations — you're an AI engineer.

The Building Block: A Single Neuron

A neuron takes multiple inputs, multiplies each by a weight, adds a bias, then applies an activation function. That's it. Everything in deep learning is billions of these stacked together.

Single Neuron Computation
z = w₁x₁ + w₂x₂ + w₃x₃ + b
output = activation(z)
w = learned weights · b = bias · activation = non-linearity 
import numpy as np

def relu(x):
    return np.maximum(0, x)  # Most common activation function

def sigmoid(x):
    return 1 / (1 + np.exp(-x))  # Squashes to 0-1 (for probabilities)

# A single neuron with 3 inputs
inputs = np.array([0.5, -0.3, 0.8])
weights = np.array([0.4, 0.7, -0.2])  # learned during training
bias = 0.1

z = np.dot(weights, inputs) + bias  # weighted sum
output = relu(z)
print(f"z = {z:.3f}")
print(f"ReLU output = {output:.3f}")

Layers and Deep Networks

Stack neurons in layers. Each layer transforms its input into a new representation. Deep networks learn hierarchical features — from simple to complex.

📥

Input Layer

Raw features enter here. One neuron per feature. No computation — just passes values forward.

🔒

Hidden Layers

The "thinking" layers. Learn increasingly abstract representations. More layers = deeper network = "deep learning".

📤

Output Layer

Produces the final prediction. Regression: 1 neuron (continuous). Classification: N neurons (one per class) with softmax.

import torch
import torch.nn as nn

# A simple feedforward neural network for digit classification
class DigitClassifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(784, 256),   # Input: 28×28 pixels flattened
            nn.ReLU(),
            nn.Dropout(0.2),       # Prevents overfitting
            nn.Linear(256, 128),   # Hidden layer
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(128, 10)     # Output: 10 classes (digits 0-9)
        )
    def forward(self, x):
        return self.network(x)

model = DigitClassifier()
print(model)
# Count parameters
total = sum(p.numel() for p in model.parameters())
print(f"Total trainable parameters: {total:,}")

Activation Functions: Why They Matter

Without activation functions, stacking layers does nothing — the result is still just a linear function. Non-linear activations let networks learn complex patterns.

FunctionFormulaRangeBest Used For
ReLUmax(0, x)[0, ∞)Hidden layers in most networks. Fast, simple, avoids vanishing gradients.
Sigmoid1/(1+e⁻ˣ)(0, 1)Binary classification output layer. Squashes to probability.
Softmaxeˣⁱ/Σeˣʲ(0, 1), sums to 1Multi-class output layer. Probability distribution over classes.
Tanh(eˣ-e⁻ˣ)/(eˣ+e⁻ˣ)(-1, 1)Hidden layers when negative outputs matter (RNNs). Zero-centered.

Backpropagation: How Networks Learn

Backprop is the algorithm that makes neural networks trainable. It efficiently computes gradients using the chain rule of calculus — flowing error signals backward through the network.

  1. Forward pass: Input flows through all layers → prediction produced.
  2. Compute loss: Compare prediction to true label (e.g., cross-entropy loss).
  3. Backward pass: Compute gradient of loss with respect to each weight using chain rule.
  4. Update weights: optimizer.step() nudges each weight to reduce loss.
import torch
import torch.nn as nn
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

# Training loop — the same pattern for ANY neural network
model = DigitClassifier()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
loss_fn = nn.CrossEntropyLoss()  # For multi-class classification

for epoch in range(10):
    for batch_images, batch_labels in train_loader:
        # 1. Forward pass
        predictions = model(batch_images.view(-1, 784))

        # 2. Compute loss
        loss = loss_fn(predictions, batch_labels)

        # 3. Backward pass (compute gradients)
        optimizer.zero_grad()  # Clear previous gradients!
        loss.backward()

        # 4. Update weights
        optimizer.step()

    print(f"Epoch {epoch+1}: Loss = {loss.item():.4f}")
💡

Common Mistake: Forgetting optimizer.zero_grad()

PyTorch accumulates gradients by default. Without zero_grad() before each backward pass, gradients from the previous step are added to the current step — completely corrupting training. Always call zero_grad() before backward().

Embeddings and Semantic Search

Transform text into mathematical meaning. Build a search engine that finds documents by concept, not just keywords.

60 min Intermediate Colab Lab 5 Quiz Questions

What Are Embeddings?

An embedding converts a piece of text into a fixed-length vector of floating-point numbers. The key property: semantically similar texts produce similar vectors.

💡

Why Vectors?

Once text is a vector, you can use linear algebra. Similarity = dot product or cosine. Clustering = k-means. Anomaly detection = outlier vectors. The entire toolkit of numerical computing becomes available for language.

from anthropic import Anthropic
# Using OpenAI embeddings (most common)
from openai import OpenAI

client = OpenAI()

def embed(text: str) -> list[float]:
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=text
    )
    return response.data[0].embedding

# Each embedding is a 1536-dimensional vector
vec = embed("What is machine learning?")
print(f"Dimensions: {len(vec)}")        # 1536
print(f"First 5 values: {vec[:5]}")    # [-0.023, 0.041, ...]

# Similar texts → similar vectors
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

v1 = embed("What is machine learning?")
v2 = embed("Explain ML to me")
v3 = embed("What is the best pizza topping?")

print(f"ML vs ML synonym: {cosine_similarity(v1, v2):.3f}")  # ~0.95
print(f"ML vs pizza:      {cosine_similarity(v1, v3):.3f}")  # ~0.30

Building Semantic Search

Keyword search matches exact words. Semantic search matches meaning. Query: "car maintenance" → finds "automobile repair guide" even though no words overlap.

import numpy as np
from openai import OpenAI

client = OpenAI()

# Knowledge base — could be thousands of documents
documents = [
    "Neural networks learn by adjusting weights through backpropagation",
    "The Transformer architecture uses self-attention to process sequences",
    "Gradient descent minimizes the loss function by following gradients",
    "Overfitting occurs when a model memorizes training data",
    "BERT and GPT are both based on the Transformer architecture",
    "Regularization techniques like dropout prevent overfitting",
    "The attention mechanism allows models to focus on relevant parts of input",
    "Cross-entropy loss is commonly used for classification tasks",
]

def embed_batch(texts):
    response = client.embeddings.create(model="text-embedding-3-small", input=texts)
    return [d.embedding for d in response.data]

# Pre-compute document embeddings (do this once, cache the results)
doc_embeddings = embed_batch(documents)

def semantic_search(query: str, top_k: int = 3) -> list[tuple[str, float]]:
    """Find the most semantically similar documents to a query."""
    query_vec = embed_batch([query])[0]
    query_arr = np.array(query_vec)

    scores = []
    for i, doc_vec in enumerate(doc_embeddings):
        doc_arr = np.array(doc_vec)
        similarity = np.dot(query_arr, doc_arr) / (
            np.linalg.norm(query_arr) * np.linalg.norm(doc_arr)
        )
        scores.append((documents[i], similarity))

    return sorted(scores, key=lambda x: x[1], reverse=True)[:top_k]

# Test it — query doesn't need to match exact words
results = semantic_search("how do models learn from mistakes?")
for doc, score in results:
    print(f"[{score:.3f}] {doc}")

Quick Check

❓ A user searches for "automobile problems". A document says "car repair guide". Will keyword search or semantic search find it?▼
Semantic search only. Keyword search requires exact word matches — "automobile" ≠ "car". Semantic search captures the meaning: automobiles = cars, problems = repair. Cosine similarity of the embedding vectors will be high for these semantically equivalent phrases.
